CSS规范 - 分类方法

CSS文件的分类和引用顺序

我们按照CSS的性质和用途，将CSS分成“基础样式”、“频道样式”、“特殊性样式”、“皮肤样式”、“单页样式”，引用时统一添加时间戳或版本号。

CSS文件引入可通过外联或内联方式引入；原则上，不允许在html上直接写样式；link和style标签都应该放入head中。

  1. 基础样式：
    ① 消除默认样式和浏览器差异(reset)。
    ② 功能性样式(f-)。
    ③ 网站通用布局(g-)。
    ④ 网站通用模块(m-)。
    ⑤ 通用元件和其扩展(u-)。
    ⑥ 皮肤类样式(s-)。
    ⑦ 其他类样式(o-)。
    ⑧ 全局特殊性样式，可用seajs异步加载单独样式。
  2. 频道样式：
    当某个频道或栏目的样式与网站整体差异较大或者维护率较高时，可以独立引用一个样式。可根据频道名称自定义设置一个前缀，建议2-5个字符。
    如有必要，可在基础样式的基础上进行样式重写。
  3. 皮肤型样式：如果产品需要换肤功能，那么我们需要将颜色、背景等抽离出来放在这里，非换肤时不可滥用。
  4. 单页样式或工程师样式：工程师负责部分可用自定义前缀，建议为姓名简称，可结合其他前缀进行组合。
  5. 其他样式。

CSS内部的分类、顺序及公示单
  1. 重置（reset）和默认（base）（mod/reset.less）：消除默认样式和浏览器差异，并设置部分标签的初始样式，以减少后面的重复劳动！
  2. 功能（function）（.f-）（mod/function.less）：为方便一些常用样式的使用，我们将这些使用率较高的样式剥离出来，按需使用。通常这些选择器具有固定样式表现，一般用于继承了父样式，但又有自身样式且样式单一时使用！级别较低，不可滥用！
  3. 布局（grid）（.g-）（mod/grid.less）：整个网站页面布局样式，频道布局可与频道前缀结合放入频道样式表，频道前缀在前（下同）。例如.ui-g-header,.ui-g-footer。

  9. JS（javascript）（.j-）：为满足条件的DOM元素绑定事件，无对应样式，仅做JS绑定。
    非j-前缀或不含自定义属性的标签不得绑定JS事件。
    ID如需绑定事件，前缀仍以此为准。
    如非绑定事件，不可滥用。
